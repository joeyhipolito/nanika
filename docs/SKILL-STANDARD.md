# Skill Standard

The definitive reference for creating, modifying, and auditing Nanika skills. One document, every convention.

**TL;DR**: A skill is a `.claude/skills/{name}/SKILL.md` directory. CLI repos own SKILL.md at `~/skills/{name}/.claude/skills/{name}/SKILL.md`; Nanika symlinks to it. `generate-agents-md.sh` extracts commands from SKILL.md into a compressed routing index in CLAUDE.md. The index is always loaded; SKILL.md is read on demand.

## 1. Architecture Overview

Nanika uses a two-layer system informed by [Vercel's research](https://vercel.com/blog/agents-md-outperforms-skills-in-our-agent-evals) showing passive context (100% pass rate) dramatically outperforms on-demand skills (53%):

| Layer | File | Role | Loaded |
|-------|------|------|--------|
| Reference | `SKILL.md` | Full command docs, examples, config | On demand |
| Routing | AGENTS-MD block in `CLAUDE.md` | Compressed index of all skills | Every turn |

All skills auto-discover from `.claude/skills/` — no flags needed.

### Directory Layout

```
~/skills/{name}/                      # CLI repo (SOURCE OF TRUTH)
├── .claude/
│   └── skills/
│       └── {name}/
│           ├── SKILL.md              # Canonical skill file
│           └── metadata.json         # Distribution metadata
├── cmd/
├── internal/
└── Makefile

~/nanika/.claude/skills/{name}/       # Symlink → ~/skills/{name}/.claude/skills/{name}
```

Each CLI skill has a companion binary: source at `~/skills/{name}/`, binary symlinked to `~/bin/{name}`. The CLI repo is the source of truth for SKILL.md — Nanika symlinks to it.

### Skill Types

| Type | Example | Location | `allowed-tools` | `## Commands` |
|------|---------|----------|----------------|--------------|
| CLI wrapper | engage, scout, orchestrator | Symlink → `~/skills/{name}/.claude/skills/{name}` | Required | Required |
| Pipeline | channels, decomposer | Real dir in `.claude/skills/` | Not required | Not required |
| Knowledge | golang-pro, vercel-react-best-practices, security-best-practices | Symlink → `~/.agents/skills/{name}` | Not required | Not required |

**CLI wrapper skills** invoke a binary and must follow the full specification in §2 (frontmatter with `allowed-tools`, `## Commands` section).

**Knowledge/reference skills** are reference documents (best practices, style guides, how-to patterns). They require only `name` and `description` in frontmatter. They must NOT include `allowed-tools`, `## Commands`, or `## Configuration` — those sections imply a runnable binary that does not exist.

**Pipeline skills** orchestrate multi-step processes within Claude's session. Like knowledge skills they omit `allowed-tools` and `## Commands`.

## 2. SKILL.md Specification

Location (canonical): `~/skills/{name}/.claude/skills/{name}/SKILL.md`

### Frontmatter

```yaml
---
name: scout
description: Gathers intelligence on configurable topics via scout CLI. Use when user asks about news, trends, scraping, intel gathering, or monitoring topics.
allowed-tools: Bash(scout:*)
argument-hint: "[topic-name]"
---
```

| Field | Required | Default | Rule |
|-------|----------|---------|------|
| `name` | Yes | — | Must match CLI binary name. Lowercase, hyphens only, max 64 chars. |
| `description` | Yes | — | Two-part: `{What it does}. Use when {trigger conditions}.` Third-person. |
| `allowed-tools` | CLI only | — | Pattern: `Bash({cli-name}:*)`. Omit for knowledge/pipeline skills. |
| `argument-hint` | No | — | Usage hint shown in slash command autocomplete. |

### Required Sections

These sections apply to **CLI wrapper skills**. Knowledge and pipeline skills have no required sections beyond a title.

| Section | Applies To | Content |
|---------|-----------|---------|
| `# {Name} — {Subtitle}` | All | One-line summary |
| `## When to Use` | CLI wrapper | Bullet list of trigger conditions (4–8 bullets) |
| `## Commands` | CLI wrapper | Bash code blocks — commands must use ` ```bash ` fences |
| `## Configuration` | CLI wrapper | Config file path: `~/.{name}/config` |
| `## Examples` | CLI wrapper | **User**/**Action** pairs mapping natural language → commands |

### Command Format Rules

Commands in `## Commands` directly determine the AGENTS-MD routing index. The script:

1. Extracts lines from ` ```bash ` blocks under `## Commands`
2. Matches lines starting with the tool name (from frontmatter `name`)
3. Falls back to directory name if frontmatter name yields no matches
4. Strips trailing `# comments` and backslash continuations
5. Takes the **first 14** commands — ordering matters

Commands must start with the CLI binary name:

```bash
scout topics                              # List all configured topics
scout gather "ai-agents"                  # Gather specific topic
scout intel "ai-agents" --json            # JSON output
```

## 3. AGENTS-MD Block Format

Auto-generated by `scripts/generate-agents-md.sh`. Never edit manually.

### Format

```
<!-- NANIKA-AGENTS-MD-START -->
[Nanika Skills Index][root: .claude/skills]IMPORTANT: ...

|{name} — {description}:{path/to/SKILL.md}|`cmd1`|`cmd2`|...|

[Domain Detection]|dev:{keywords}|personal:{keywords}|...
[Orchestration Triggers]|keywords:{invoke: orchestrator run}|
[Structure]|skills:{.claude/skills}|docs:{./docs}|
<!-- NANIKA-AGENTS-MD-END -->
```

### Generation

```bash
./scripts/generate-agents-md.sh              # Generate + inject
./scripts/generate-agents-md.sh --dry-run    # Print without writing
./scripts/generate-agents-md.sh --diff       # Show what would change
```

## 4. Distribution

CLI repos are the unit of distribution. Each repo at `~/skills/{name}/` owns its SKILL.md and metadata.json.

### Installation

```bash
npx skills i owner/skill-name          # Install from GitHub
cp -r .claude/skills/scout ~/.claude/skills/  # Manual copy
```

### Cross-Agent Compatibility

| Agent | Skill Location |
|-------|---------------|
| Claude Code | `~/.claude/skills/{name}/SKILL.md` |
| Gemini CLI | `.gemini/skills/{name}/SKILL.md` |

## 5. Checklists

### New Skill

```
□ 1.  Create CLI: ~/skills/{name}/cmd/{name}-cli/main.go + Makefile
□ 2.  Create SKILL.md in CLI repo: ~/skills/{name}/.claude/skills/{name}/SKILL.md
□ 3.  Symlink into Nanika: ln -s ~/skills/{name}/.claude/skills/{name} .claude/skills/{name}
□ 4.  Regenerate routing index: ./scripts/generate-agents-md.sh
□ 5.  Verify skill appears in CLAUDE.md AGENTS-MD block
```

### Update Skill

```
□ 1. Update SKILL.md in CLI repo (symlink propagates to Nanika)
□ 2. Regenerate routing index: ./scripts/generate-agents-md.sh
□ 3. Verify AGENTS-MD block in CLAUDE.md
```

## 6. Conventions

### Naming

- Skill directory name = CLI binary name = SKILL.md frontmatter `name`
- **Exception**: orchestrator's skill is named `missions` (SKILL.md `name: missions`)

### Standards Compliance

This system follows the [Agent Skills Standard](https://agentskills.io). Compatible with Anthropic (YAML frontmatter, allowed-tools), Vercel Labs ([skills CLI](https://github.com/vercel-labs/skills)), and Cloudflare RFC.
